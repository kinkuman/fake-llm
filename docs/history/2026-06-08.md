## 概要

`fake-llm` は、LLM のふりをする小さなローカル用おもちゃサーバとして作成した。Chat Completions 互換風の API で呼び出せるが、中身はルールベースの人工無能であり、本物の LLM ではない。

## 方針

- 名前は `fake-llm` とし、ジョーク用途とローカルデバッグ用途が伝わる形にする。
- 画像や GUI は含めず、CLI と HTTP API に絞る。

## 返答ロジック

現在の主な返答モードは次の通り。

- `PatternResponder` 相当
  - `pattern.tsv` の正規表現に一致したら、その行の返答候補からランダムに返す。
- `RandomResponder` 相当
  - `random.txt` からランダムに返す。
- `WhatResponder` 相当
  - `入力ってなに？` の形で聞き返す。
- Markov 連鎖
  - 会話中の単語のつながりをオンメモリで学習する小さな実装を入れている。
  - 現在は通常返答経路では強く使っていない。
- Template
  - `template.tsv` を読む実装は残っている。
  - 現在は確率分岐を優先したため、通常返答経路からは外している。


実際の意味。

- 60%: pattern
  - pattern に一致しない場合は random に落とす。
- 30%: random
- 10%: what

## 注意点

- `pattern.tsv` は TSV なので、pattern と response の間はスペースではなく TAB が必要。
- VS Code が TAB をスペースに変換する場合、`editor.renderWhitespace` や `[tsv] editor.insertSpaces = false` の設定が有効。
- API は互換風の最小実装であり、本物の API を完全再現していない。
- API key は任意文字列でよい。認証はしていない。

## 追加作業: 1語発話の文字分解学習を廃止

Janome 導入直後は、`こんにちは` のような1語発話も Markov に入れるため、長い1トークンを文字単位へ分解していた。

しかし、文字単位の chain が混ざると Markov の品質が落ちるため、この補助処理は削除した。今後は3トークン未満の発話は Markov 学習しない。

例。

```text
こんにちは -> 学習しない
りんごとみかんを食べる -> ['りんご', 'と', 'みかん', 'を', '食べる'] として学習する
```

既存の `state.json` には文字単位 chain が混ざっていたため、`state.char.json` に退避した。新しい会話から作られる `state.json` は単語単位寄りになる。

## 追加作業: 感情モデルを追加

本家 `nobypy/markov` の `Emotion` に近い、小さな `mood` モデルを追加した。

変更点。

- `PatternRule` を追加し、`pattern.tsv` の1行を object として扱うようにした。
- `Emotion` を追加。
  - `mood_min = -15`
  - `mood_max = 15`
  - `mood_recovery = 0.5`
- `FakeLLM` が `self.emotion` を持つようにした。
- `reply()` のたびに、入力に一致した pattern の `mood_delta` を mood に反映するようにした。
- mood は毎回少しずつ0へ戻る。
- `state.json` に `mood` も保存・復元するようにした。
- `pattern.tsv` の3列目に任意の `mood_delta` を追加できるようにした。

`pattern.tsv` の形式。

```text
regex<TAB>response|response<TAB>mood_delta
```

3列目は省略可能。省略時は `0`。

例。

```text
ありがとう|助かった<TAB>どういたしまして<TAB>2
バカ|ばか|馬鹿<TAB>だって無能ですもん<TAB>-2
```

## 追加作業: mood 条件付き返答を追加

本家 `PatternItem.choise(mood)` に近づけるため、返答候補に `need##phrase` を書けるようにした。

形式。

```text
regex<TAB>need##phrase|need##phrase<TAB>mood_delta
```

`need` の意味。

- `0` または省略: 常に候補になる。
- 正の値: 現在の mood がその値より大きい時だけ候補になる。
- 負の値: 現在の mood がその値より小さい時だけ候補になる。

例。

```text
愛してる<TAB>0##はいはい、そうですか、次へいきましょう。|3##急に優しいですね|6##調子が狂いますね<TAB>3
バカ|ばか|馬鹿<TAB>0##だって無能ですもん|-5##もう少しやさしく話してくれませんか<TAB>-2
```


```text

```

## 追加作業: state の肥大化対策

`state.json` は長期記憶ではなく短期記憶として扱う方針にした。
SQLite はまだ不要と判断し、JSON のまま保存件数に上限を付けた。

上限値。

- 学習 random 返答: 最新50件
- 学習 pattern ルール: 最新100件
- 学習 template: キーワード数ごとに最新30件
- Markov の開始候補: 最新200件
- Markov の prefix: 最新400件
- Markov の suffix: prefixごとに最新20件
